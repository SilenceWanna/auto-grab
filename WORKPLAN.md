# 12306 自动抢票脚本 — 工作计划

> 本项目是一个运行在 PC 端的 12306 自动抢票工具，采用**浏览器自动化**方案（DrissionPage 驱动真实浏览器）。
> 本文档为完整的开发路线图与实现说明。

---

## 一、项目目标

实现一个可在个人电脑上运行的自动化脚本，能够：

1. 自动登录 12306 账号并保持会话。
2. 按用户配置的车次、日期、席别、乘客，轮询查询余票。
3. 一旦出现余票，自动完成选座与提交订单，占座成功。
4. 抢票成功后通过声音 / 桌面通知 / 消息推送提醒用户尽快支付。

> ⚠️ **合规声明**：本项目仅供个人学习与自用，用于代替手动刷新抢票。请勿用于黄牛倒票、高频请求攻击 12306 服务器等违规用途。使用者需对自身行为负责，遵守 12306 的服务条款与相关法律法规。

---

## 二、技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| 语言 | Python 3.10+ | 生态成熟、开发效率高 |
| 浏览器自动化 | [DrissionPage](https://github.com/g1879/DrissionPage) | 兼具 selenium 的页面控制与 requests 的接口能力，反爬能力强，无需单独管理 driver |
| 配置管理 | YAML | 可读性好，便于用户手动填写车次/乘客信息 |
| 日志 | logging | 标准库，分级记录抢票全过程 |
| 通知 | 声音 + 桌面通知 + 可选 Server酱/钉钉 Webhook | 多渠道确保用户及时收到 |
| 打包（可选） | PyInstaller | 后期可打包为 exe，方便非技术用户使用 |

**为什么选浏览器自动化而非直接调 HTTP 接口**：12306 的接口带有设备指纹、接口签名、滑块验证码等强反爬手段，直接逆向接口极易失效且维护成本高。浏览器方案对页面改版鲁棒性更强，登录与验证码环节可人工介入，更适合个人长期使用。

## 五、关键技术难点与对策

| 难点 | 对策 |
|------|------|
| 验证码 | 优先登录阶段人工过一次并复用 cookies；预留第三方打码平台接口 |
| 反爬 / 频控 | 轮询间隔加随机抖动；使用真实浏览器指纹；控制请求频率 |
| 页面改版导致选择器失效 | 选择器集中管理、易于维护；关键步骤加显式等待与失败重试 |
| 放票瞬间高并发 | 提前登录占位、预填订单信息，命中后最快路径提交 |
| 长时间运行稳定性 | 全流程异常隔离 + 自动重试 + 心跳日志 |

## 六、使用风险提示

- 12306 页面结构与风控策略可能随时变化，脚本需要持续维护。
- 抢票成功不等于支付成功，务必及时人工确认支付。
- 请合理设置轮询频率，避免对 12306 造成压力或触发封控。
